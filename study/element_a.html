<address>

  • 문서나 글의 저자 또는 회사와 연락할 수 있는 정보를 명시할 때 사용한다.
  • body 안에 address는 해당 문서의 연락 정보를 나타내며, article 안의 address는 해당 글에 대한 연락 정보를 나타낸다.
  • address는 주로 italic체로 표현되며, address 요소의 위쪽과 아래쪽에 약간의 공백을 자동으로 삽입하여 다른 텍스트와 구분한다.
  • HTML 4에선 article을 지원하지 않기 때문에 언제나 해당 문서의 연락 정보만을 나타낸다.
<address>서울시 구로구 구로동</address>

<applet>

  • HTML4에서 사용되었다. HTML5에서 사용되지 않는다.
  • 인터넷 익스플로러 11버전과 그 이전 버전에서는 플러그인(plug-in)을 사용하여 지원하고 있으며, 그 외 몇몇 브라우저들에서도 플러그인이나 기타 프로그램을 설치함으로써 요소를 사용할 수 있도록 지원하고 있습니다.
  • 대체할 태그로 동영상을 넣으려면 video, 음악을 넣으려면 audio, 개체를 넣으려면 embed, object를 사용한다.

aria-label & aria-labelledBy

  • aria-label은 화면에 현재 요소를 설명할 텍스트가 없을 경우에 사용하는 설명용 텍스트를 담고있고
  • aria-labelledby는 화면에 현재 요소를 설명할 텍스트가 있을 경우에 해당 텍스트 영역과 현재 요소를 연결할 때 사용한다.
<button class="btnMenu" aria-label="navigation menu">
  • 예를들어, 햄버거 메뉴 버튼이 있다고 한다면 일반적인 사용자들의 경우 햄버거 메뉴 이미지를 보고 클릭을 하고 사용하겠지만,
    시각적으로 햄버거 메뉴 이미지를 확인할 수 없는 사용자들의 경우,
    해당 영역이 어떤 버튼인지 인지 할 수 없다. 그렇기 때문에 aria-label 속성을 사용해 해당 영역이 navigation menu 라는 정보를 담아준다.

<audio>

  • 음악이나 오디오 스트림과 같은 사운드를 정의할 때 사용
  • <audio> 요소 내에 위치하는 텍스트는 사용자의 브라우저가 </audio> 요소를 지원하지 않을 경우 화면에 표시됩니다.
  • 현재 다음 세 가지 포맷의 파일을 지원하고 있다.
    • MP3 : Moving Picture Experts Group에 의해 개발되었으며, MPEG-1의 오디오 규격으로 개발된 손실 압축형 파일 형식이다.
      모든 브라우저에서 다 가능하다.
    • WAV : IBM과 Microsoft에 의해 개발되었으며, 개인용 PC에서 오디오를 재생하기 위한 IBM과 Microsoft의 표준 오디오 파일 형식이다.
      ie에서 지원하지 않는다.
    • Ogg : Xiph 재단에 의해 개발되었으며, MP3의 대안으로 개발된 특허권으로 보호되지 않는 개방형 공개 멀티미디어 파일 형식이다.
      ie와 safari에서 지원하지 않는다.
  • audio 옵션
    • controls : 오디오 재생, 볼륨, 탐색, 일시 정지 컨트롤을 브라우저에서 제공
    • currentTime : 오디오 재생의 현재 위치(초)를 설정하거나 반환. 설정하면 재생이 지정된 위치로 이동. ie에서 지원하지 않는다.
    • autoplay : 자동 재생
    • loop : 오디오가 끝나면 자동 재생
    • muted : 음소거
    • preload : 페이지가 로드될 때 오디오 파일이 같이 로드되어야 하는지 여부와 그 방법을 명시
    • src : 오디오 파일의 위치
<figure>
    <figcaption>sample_audio_ogg 파일</figcaption>
    <audio controls>
        <souce src="examples/media/sample_audio_ogg.ogg"></souce>
        <souce src="examples/media/sample_audio_ogg.mp3"></souce>
        <souce src="examples/media/sample_audio_ogg.wav"></souce>
        이 문장은 브라우저가 audio 태그를 지원하지 않을 때 화면에 표시된다.
    </audio>
</figure>
