<canvas>

  • Canvas API나 WebGL API를 사용하여 그래픽이나 애니메이션을 렌더링
    • Canvas API : Javascript나 canvas 태그를 이용해 그래픽을 그리기 위한 수단이다. 애니메이션, 게임 그래픽, 데이터 시각화, 사진 조작 및 실시간 비디오 처리를 위해 사용
    • WebGL API : 2D 및 3D 그래픽을 그린다.
    • ctx : context를 의미
    • document.getElementById : id 속성을 가진 요소를 찾는다.
    • HTMLCanvasElement.getContext() : 엘리먼트의 컨텍스트(렌더링 될 그리기의 대상)을 얻는다.
    • CanvasRenderingContext2D : 인터페이스를 사용해 수행한다. 드로잉면에 대한 2 차원 렌더링 컨텍스트 제공. 도형, 텍스트, 이미지 및 기타 개체를 그리는 데 사용됩니다.
    • fillStyle : 캔버스에 그려져 있는 도형을 스타일로 채운다.
    • createLinearGradient : 시작점과 끝점의 좌표만 지정한다. 시작점에서 끝점으로 색상이 점점 변하는데 두 점의 위치에 따라 무늬의 방향이 결정된다. 선형 그라데이션 지정(시작점 x1, 시작점 y1, 끝점 x2, 끝점 y2)
    • createRadialGradient : 6개의 매개변수로 지정된다. 원형 그라데이션 지정(시작하는 원의 x축 좌표, 시작하는 원의 y축 좌표, 시작하는 원의 반지름, 마지막 원의 x축, 마지막 원의 y축, 마지막 원의 반지름)
    • addColorStop(offset, color) : offset은 0~1 사이의 실수값으로 색상의 위치를 지정한다. 0은 시작점, 1은 끝점, 0.5은 중간점이다.
<canvas id="canvasRect" width="200" height="100" style="border:1px solid #000000;"></canvas>

    <canvas id="canvasStrokeText" width="200" height="100" style="border:1px solid #000000;"</canvas>
    <canvas id="canvasLineGradient" width="200" height="100" style="border:1px solid #000000;"</canvas>
    <canvas id="canvasCircleGradient" width="200" height="100" style="border:1px solid #000000;"</canvas>
    <script>
        const canvas = document.getElementById('canvasRect');
        const canvas2 = document.getElementById('canvasCircle');
        const canvas3 = document.getElementById('canvasStroke');
        const canvas4 = document.getElementById('canvasFillText');
        const canvas5 = document.getElementById('canvasStrokeText');
        const canvas6 = document.getElementById('canvasLineGradient');
        const canvas7 = document.getElementById('canvasCircleGradient');
        const ctx = canvas.getContext('2d');
        const ctx2 = canvas2.getContext('2d');
        const ctx3 = canvas3.getContext('2d');
        const ctx4 = canvas4.getContext('2d');
        const ctx5 = canvas5.getContext('2d');
        const ctx6 = canvas6.getContext('2d');
        const ctx7 = canvas7.getContext('2d');

        
        ctx.fillStyle = 'green';
        ctx.fillRect(10, 10, 150, 100); //x, y, width, height

        
        ctx2.beginPath(); //새로운 선을 그리겠다는 선언
        ctx2.arc(95, 50, 50, 0, 2 * Math.PI); //(x, y, 반지름, 시작 각도, 마지막 각도 * 원주율)
        ctx2.stroke();

        
        ctx3.moveTo(0, 0); //시작 좌표로 시작점을 나타냄
        ctx3.lineTo(200, 100); //끝나는 좌표
        ctx3.stroke();

        ctx4.font = "30px Arial" //폰트 정의 = "사이즈, 패밀리"
        ctx4.fillText("Hello", 10, 50);

        ctx5.font = "30px Arial"
        ctx5.strokeText("Hello", 10, 50); //("문자", x, y)

        const grad = ctx6.createLinearGradient(0, 0, 200, 0);
        grad.addColorStop(0, "red");
        grad.addColorStop(1, "white");
        ctx6.fillStyle = grad;
        ctx6.fillRect(10, 10, 150, 80);
    </script>

HTMLCanvasElement.width (en-US) canvas 요소에서 HTML 속성인 width를 반영하며, css의 pixel 값으로 해석된다. 속성값이 없거나 음수일 경우, 기본값인 300이 주어진다.
HTMLCanvasElement.height (en-US) canvas 요소에서 HTML 속성인 height를 반영하며, css의 pixel 값으로 해석된다. 속성값이 없거나 음수일 경우, 기본값인 150이 주어진다.
HTMLCanvasElement.mozOpaque (en-US) 표준 X HTML 속성인 moz-opaque(투명도가 중요한 요소인지를 알려준다. Mozila 기반 브라우저에서만 지원하므로, 표준 canvas.getContext('2d', { alpha: false })를 사용한다.)를 반영하는 객체. 반투명이 인자로 사용될지에 대한 여부를 알려준다. 반투명 요소가 없다면, canvas 성능이 최적화될 수 있다.
HTMLCanvasElement.mozPrintCallback 표준 X 페이지가 프린트 되는 경우 호출되는 함수. 만약 프린터가 사용되는 경우 canvas를 더욱 고해상도로 다시 그리게 할 수 있습니다.
CanvasPattern CanvasRenderingContext2D.createPattern()방법으로 만든 이미지, 캔버스 또는 비디오를 기반으로 패턴을 설명한다. fillStyle 또는 strokeStyle을 사용할 수 있다. ie 지원 불가
CanvasPattern.setTransform() 패턴에 선형 변환을 적용 SVGMatrix(더이상 사용하지 않는다.) 하거나 DOMMatrix(회전과 번역을 포함하여 2D 및 3D 작업에 적합한 4 x 4로 나타낸다.) 나타낸다. 일부 구현에서는 아직 허용하지 않는다.

HTMLCanvasElement.captureStream() 실시간 비디오로 캡처할 수 있는 CanvasCaptureMediaStream (en-US)을 반환한다. ie 불가
ex) MediaStream = canvas.captureStream(frameRate); //frameRate : 각 프레임의 캡처 속도를 나타내는 배정밀도 부동 소수점 값. 설정하지 않으면 캔버스가 변경될 때마다 새 프레임이 캡처된다.
const stream = canvasElt.captureStream(25);
CanvasCaptureMediaStreamTrack 수동으로 프레임을 캡처하여 스트림으로 보내도록 한다.
HTMLCanvasElement.toDataURL() (en-US) type 파라미터에서 지정하고 있는 포맷(기본: png) 의 이미지를 나타내는 data-URL을 반환, 반환된 이미지는 96dpi의 해상도를 갖는다.
HTMLCanvasElement.toBlob() (en-US) 캔버스가 포함하고 있는 이미지를 나타내는 Blob 객체를 생성.
HTMLCanvasElement.transferControlToOffscreen() (en-US)
HTMLCanvasElement.transferControlToOffscreen() (en-US) 캔버스에 오브젝트를 그리는 데엔 여러 path와 드로잉 커맨드가 쓰이는데, Path2D()를 써서 간단하게 만들 수 있다.
ex) const rectangle = new Path2D(); rectangle.rect(10, 10, 45, 45);
CanvasGradient 그래디언트를 할 수 있게 해주는 기능. 모든 브라우저 사용 가능
CanvasGradient.addColorStop() : offset 및 a로 정의된 새 정지점 color을 그래디언트에 추가한다.
오프셋이 0 ~ 1까지 설정 가능하며, 그 외의 숫자나 음수를 설정하면, INDEX_SIZE_ERR가 발생한다.
색상을 CSS로 구문 분석할 수 없으면 SYNTAX_ERR가 발생한다.
ImageData canvas 엘리먼트의 픽셀 데이터를 나타낸다. 인자로 주어진 Uint8ClampedArray(0 - 255 클램핑 8 비트 부호없는 정수의 배열을 나타내는 입력; [0,255] 범위를 벗어난 값을 지정하면 0 또는 255가 대신 설정됩니다. 정수가 아닌 값을 지정하면 가장 가까운 정수가 설정되며, 내용은 0 으로 초기화 된다.)
로 부터 해당 크기에 맞는 ImageData객체를 생성한다.
만약 인자가 주어지지 않으면 검정색 사각형 이미지를 생성
TextMetrics 현재 글꼴 스타일로 캔버스에 그려진 텍스트의 너비와 같은 숫자 정보를 검색한다.

<caption>

  • 테이블, 사진, 삽화 등에 설명 혹은 제목을 정의할 때 사용한다.
  • table 요소의 caption은 하나만 사용할 수 있으며, table 태그 바로 다음에 사용해야 한다.
  • css을 사용해서 위치나 정렬 방법을 변경할 수 있다.
<table> 
<caption>색상의 종류에 대한 테이블 입니다.</caption> 
    <tr>
        <th scope="col">레드</th>
        <th scope="col">화이트</th>
        <th scope="col">블랙</th>
    </tr> 
</table>

<colgroup> & <col>

  • colgroup
    • 테이블에서 서식 지정을 위해 하나 이상의 열을 그룹으로 묶을 때 사용한다.
    • 행(row)이나 셀(cell)의 스타일을 반복하지 않고, 열(column) 전체에 다른 스타일을 적용하고 싶을 때 유용하게 사용할 수 있다.
    • 테이블 요소의 자식 요소로, 모든 caption 요소보다 뒤에 위치해야 하며 모든 thead, tbody, tfoot, tr 요소보다는 앞에 위치해야 한다.
  • col
    • colgroup 요소에 속하는 각 열의 속성을 정의할 때 사용한다.
    • 각 행(row)이나 셀(cell)의 스타일을 반복하지 않고, 열마다 다른 스타일을 적용하고 싶을 때 사용할 수 있다.
    • HTML에서는 태그를 닫지 않지만, XHTML에서는 다음과 같이 반드시 태그를 닫아야만 한다.

필수 속성
속성명 속성값 설명
align left right center justify char 콘텐츠의 정렬 방식. HTML5에서는 더 이상 지원하지 않음. 예) <col align="left">
char 문자 콘텐츠의 정렬에 사용되는 문자(character)를 명시함. HTML5에서는 더 이상 지원하지 않음. 예) <col align="char" char=".">
charoff 숫자 char 속성에서 명시된 문자를 기준으로 정렬될 콘텐츠의 문자 수를 명시함. HTML5에서는 더 이상 지원하지 않음. 예) <col align="char" char="." charoff="2">
span 숫자 합쳐질 열(column)의 개수를 명시함. HTML5에서는 더 이상 지원하지 않음. 예) <col width="20%">
valign top middle bottom baseline 관련된 콘텐츠의 수직 정렬 방법을 명시함. HTML5에서는 더 이상 지원하지 않음. 예) <col valign="top">
width % 픽셀 상대 길이 요소의 너비를 명시함. HTML5에서는 더 이상 지원하지 않음.
때문에 인라인 스타일로 제어를 해야 한다.
예) <col style="width:20%">

colgroup에 인라인 스타일로 배경색을 넣으면 테이블 전체에 색상이 입혀진다.

col에 인라인 스타일로 배경색을 넣으면 해당 컬럼 전체에 색상이 입혀진다.
