<b>

  • 단순히 텍스트를 굵게 표현하기 위해 사용된다.
  • HTML5에서 강조된 텍스트는 em, 중요한 의미를 가지는 텍스트는 strong, 하이라이트 된 텍스트는 mark을 써준다.
  • 위의 태그들에서 적절한 태그를 찾을 수 없을 때만 b 태그를 쓴다.
  • b 요소는 콘텐츠의 중요성 보다는 텍스트 자체에 주의를 끌기 위해 사용되지만, strong은 콘텐츠 자체의 중요성을 강조하기 위해 사용된다.
<p>
    <b>단순히 굵게 표현된 텍스트</b>
</p>

<base>

  • 해당 문서의 모든 상대 주소(relative URL)에 대한 기본 URL(base URL)과 target 속성값을 정의할 때 사용
  • href 속성 또는 target 또는 둘 모두를 가져야 한다.
  • 하나의 문서에 최대 하나만 존재할 수 있고, head 요소 내에 위치해야 한다.
  • body 요소 내에 쓰게 될 경우, body 안에 있는 href, src들이 에러가 난다.
<head>
    <base href="/images/example.png"> 
</head>

<body>
    <img src="images.png" alt="">
</body>

<bdi>

  • Bi-Directional Isolation의 줄임말이며, 주위의 텍스트들과는 다른 방향으로 써지는 텍스트 영역을 정의할 때 사용
  • 한국어, 영어는 왼쪽에서 오른쪽이지만, 히브리어나 아랍어 등과 같이 오른쪽에서 왼쪽으로 흐르는 언어가 필요할 때 사용하는 요소로 독립적인 양방향 텍스트 영역을 정의할 때 사용
  • 자동으로 텍스트 방향을 제어하다 보니, 해석 오류로 인해 비정상적으로 출력 될 수 있다. dir="auto" 설정은 텍스트의 방향성을 정말로 알 수가 없으므로, 서버 쪽에서 사용할 수 있는 더 좋은 알고리즘이 없을 때 최후의 보루로 사용하기를 권한다.
<p><bdi>test</bdi></p> (영문 : 영향 없음)
<p><bdi>إيان</bdi></p> (아랍어 : 오른쪽 → 왼쪽)

<bdo>

  • 텍스트 양방향 제어를 무시하고 텍스트 방향을 지정할 수 있게 된다.
  • 현재의 텍스트 방향을 재정의 하는 데 사용
  • bdo의 속성인 dir 속성의 기본값은 부모 요소로부터 상속받지 않는다는 의미를 가지는 auto 값으로 자동 설정된다.
<p><bdo dir="rtl">test</bdo></p>

<blockquote>

  • 다른 소스에서 인용 된 부분을 지정한다.
  • 일반적으로 blockquote 요소를 들여쓴다. 들여쓰기를 하고 싶지 않으면 css에 margin-left:0;을 주면 된다.
  • blockquote 요소는 대부분의 브라우저에서 아래 css를 기본값으로 표시한다.
<blockquote>
    HTML은 하이퍼텍스트 마크업 언어(HyperText Markup Language, 문화어: 초본문표식달기언어, 하이퍼본문표식달기언어)라는
    의미의 웹 페이지를 위한 지배적인 마크업 언어다. HTML과 CSS 표준의 공동 책임자인 W3C는 명확하고 표상적인 마크업을 위하여
    CSS의 사용을 권장한다.      
 </blockquote>
blockquote{
    display:block;
    margin-top:1em;
    margin-bottom:1em;
    margin-left:40px;
    margin-right:40px;
}

<br>

  • break의 약자
  • 강제로 줄바꿈을 할 때 사용
  • br 태그를 사용할 땐 주의해서 써야 한다.. 반응형 디자인일 땐 사용을 권하지 않는다. UI 상 봤을 때 텍스트가 고정적으로 있을 때 권장한다.
  • HTML4에선 <br>만 허용하고 HTML5에선 <br> 선호, <br>/ 및 <br> / 사용 가능하고 XHTML에선 <br /> 선호, <br></br>을 사용 가능하다.

<button>

  • 클릭 버튼을 정의한다.
  • button 요소는 i, b, strong, br, img 등을 넣을 수 있다.
  • button 요소에 type을 넣어서 button이 어떠한 기능을 하는지 알려준다.
  • button type에 submit의 기능은 서버로 데이터를 전달하는 역할을 한다.
  • button type에 reset의 기능은 form 데이터의 입력된 데이터를 초기화 한다.
<button type="button">Button</button>
<button type="submit" value="내용전달">Submit</button>
<button type="reset" value="내용취소">Reset</button>
