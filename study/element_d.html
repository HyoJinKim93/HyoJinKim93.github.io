<dl> & <dt> & <dd>

  • dl
    • 용어와 그에 대한 설명을 리스트 형식으로 정의할 때 사용한다.
    • 용어나 이름을 나타내는 dl과 그에 대한 설명 dd로 구성된다.
  • dt
    • 설명 목록의 용어 / 이름을 정의한다.
    • dl의 하위 태그이고 설명을 하는 dd과 함께 구성된다.
  • dd
    • 설명 목록의 용어 / 이름을 설명한다.
    • dl의 하위 태그이고 용어 / 이름을 뜻하는 dt와 함께 구성된다.
<dl>
    <dt>description list</dt>  
    <dd>특정 용어나 단어를 설명하는 리스트</dd> 
</dl>

<data>

  • 데이터 프로세스에 대해 기계가 읽을 수 있는 값과 브라우저에서 렌더링 하기 위해 사람이 읽을 수 있는 값을 모두 제공한다.
  • 시간 혹은 날짜 관련 정보라면 data 대신 <time> 요소를 사용한다.
  • HTML5에서 새로 추가된 엘리먼트
<ul>
    <li data value="398">Data1</li>  
    <li data value="399">Data2</li>  
    <li data value="400">Data3</li>  
</ul>

상품의 이름을 표시하면서, 각각의 상품 번호도 연결한다.

<datalist>

  • input 요소에 데이터를 입력할 때 미리 정의된 옵션을 드롭다운 리스트로 보여줌으로써 자동 완성 기능을 제공한다.
  • input 요소의 list 속성값으로 datalist 요소의 id 값을 주면, 해당 datalist 요소에서 미리 정의한 옵션들을 input 요소에서 사용할 수 있다.
<form action="" method="get">
    나라 : <input type="text" name="country" list="country"> 
    <datalist id="country">
        <option value="KR">
        <option value="USA">
        <option value="FR">
    </datalist>
</form>

<details>

  • 텍스트 한가운데 라인을 추가하여 문서에서 삭제된 텍스트를 표현할 때 사용한다.

<details>
    <summary>오늘의 날씨</summary>
    <ul>
        <li>미세먼지 : 좋음</li>
        <li>초미세먼지 : 좋음</li>
        <li>강수확률 : 30%</li>

  • 미세먼지 : 좋음
  • 초미세먼지 : 좋음
  • 강수확률 : 30%
  • details 요소를 이용해서 accordion을 만들 수 있지만, detail은 ie, edge에서 지원을 하지 않는다.

  • details 태그 안에 summary 태그가 없으면 「세부정보」로 기본 요약 키워드가 자동 표시된다.

<dfn>

  • HTML에서 용어를 정의할 때 사용
  • dfn 요소의 상위에는 해당 용어에 대한 정의나 설명이 반드시 있어야 한다.
  • dfn 요소가 title 특성을 가지고 있으면 그 값을 현재 정의 중인 용어로 간주한다.
  • dfn은 여전히 텍스트 콘텐츠를 가지고 있겠지만, 완전한 용어 대신 준말(abbr)을 넣을 수도 있고, 다른 대체 형태일 수도 있다.
  • dfn이 다른 텍스트는 없이 하나의 자식만 가지는데, 그 자식이 title 특성을 가진 abbr인 경우, 자식 abbr의 title 특성 값을 현재 용어로 간주한다.
  • dfn 요소에 id를 지정하면, a 요소로 dfn을 가리킬 수 있다. 사용자가 용어 뜻을 잘 모를 경우, 링크를 클릭해 확인할 수 있다.

<dialog>

  • 닫을 수 있는 경고, 검사기, 창 등 대화 상자 및 기타 다른 상호작용 가능한 컴포넌트를 나타낸다.
  • 쉽게 웹 페이지에서 팝업 대화 상자를 만들 수 있도록 해준다.
  • 대화형 콘텐츠 요소는 aria가 관리하기 때문에 tabindex 특성을 dialog 요소에 사용해서는 안 된다.
  • method="dialog" 특성을 사용한 form 요소는 제출 시 닫는다.
<dialog>
    <p>This is dialog<p>
</dialog>
