<font>

  • HTML4에서는 font face, font size, color를 지원한다.
  • HTML에서 <font>은 실제 크기가 작다.
  • font 태그는 HTML5에서 지원하지 않는다. CSS로 처리하는 걸 추천
<font>HTML5에서 더 이상 지원하지 않으며, CSS로 처리하는 걸 추천합니다.</font>

<form>

  • 사용자 입력을 위한 HTML 폼을 생성하는 데 사용된다.
  • <form>에서 사용하는 태그들은 다음과 같다.
    <input>
    <textarea>
    <button>
    <select>
    <option>
    <optiongroup>
    <fieldset>
    <label>
    <output>
  • HTML5에서는 form 요소에 autocomplete 속성과 novalidate 속성이 새롭게 추가되었고, accept 속성은 더 이상 지원되지 않도록 변경
  • XHTML에서는 name 속성을 더 이상 지원하지 않으므로, name 속성 대신 전역 속성인 id 속성을 사용해야 한다.
  • form 중, valid, invalid 부분은 css :valid와 :invalid pseudo-class를 사용할 수 있다.
<form action="" method="get">
    <label for="name">First name: <>
    <input type="text" id="name">
    <button type="submit">Submit</button>
</form>
필수 속성
속성명 속성값 설명
accept 파일 타입 서버로 업로드 할 수 있는 파일 타입을 콤마로 구분된 리스트로 명시.
HTML5에선 더 이상 지원 안 함.
accept-charset 문자셋 폼 데이터(form data)를 서버로 보낼 때 사용되는 문자 인코딩(character encoding) 방식을 명시
accept-charset 속성의 기본값은 “UNKNOWN”으로 설정한다. <form accept-charset="문자셋"> 참고 : http://tcpschool.com/html-tag-attrs/form-accept-charset
action URL 폼 데이터가 서버로 제출될 때 해당 데이터가 도착할 URL을 명시함.
enctype

application/x-www-form-urlencoded : 기본값으로, 모든 문자들은 서버로 보내기 전에 인코딩 됨을 명시

multipart/form-data : 모든 문자를 인코딩하지 않음. 주로 파일이나 이미지를 서버로 보낼 때 주로 사용

text/plain : 공백 문자(space)는 "+" 기호로 변환하지만, 나머지 문자는 모두 인코딩되지 않음을 명시함.

 폼 데이터가 서버로 제출될 때 해당 데이터가 인코딩되는 방법을 명시함.
단, form 요소의 method 속성이 post인 경우에만 가능
<form action="/home/uploadfiles" method="post" enctype="multipart/form-data"></form> 참고 : http://tcpschool.com/html-tag-attrs/form-enctype
autocomplete on, off form 요소에서 자동 완성 기능을 사용할지 여부를 명시함.
method
  • GET
    • URL에 폼 데이터를 추가하여 서버로 전달하는 방식이다.
    • HTTP 요청은 브라우저에 의해 캐시되어 저장된다.
    • 보통 쿼리 문자열에 포함되어 전송되므로, 길이의 제한이 있다. 따라서 보안상 취약점이 존재하므로, 중요한 데이터는 POST 형식으로 하는 것이 좋다.
  • POST
    • 폼 데이터를 별도로 첨부하여 서버로 전달하는 방식이다.
    • HTTP 요청은 브라우저에 의해 캐시되지 않고, 히스토리에도 남아있지 않는다.
    • HTTP 요청에 의한 데이터는 쿼리 문자열과는 별도로 전송된다.
    • 데이터의 길이에 제한이 없고, 보안성이 높다.
 폼 데이터가 서버로 제출될 때 사용되는 HTTP 메소드를 명시함.
name 텍스트 form 요소의 이름을 명시함.
novalidate novalidate 폼 데이터가 서버로 제출될 때 해당 데이터의 유효성을 검사하지 않음을 명시함.
target _blank, _self, _parent, _top 폼 데이터를 서버로 제출한 후 받는 응답이 열릴 위치를 명시함.

지원 지원 지원 지원 지원 지원

<frame>

  • framset 요소에 포함되는 하나의 프레임을 정의할 때 사용한다.
  • framset 요소는 border, 스크롤, 크기 재조정 가능 여부 등 서로 다른 속성값을 가진다.
  • HTML5에선 지원하지 않는다.
    더 이상 지원되지 않는 frame 요소를 사용하면서 HTML 유효성 검사를 통과하기 위해서는 다음의 두 가지 방법처럼 DOCTYPE을 선언해야 한다.
    <!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/html4/loose.dtd">
    <!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Frameset//EN" "http://www.w3.org/TR/html4/frameset.dtd">
  • HTML에서는 frame 태그를 닫지 않지만, XHTML에서는 태그를 반드시 닫아야 한다.
<frameset cols="25%,*,25%">
    <frame name="left" src="/ex1/intro" />
    <frame name="center" src="/ex2/intro" />
    <frame name="right" src="/ex3/intro" />
</frameset>

속성명 속성값 설명

HTML5에서는 더 이상 지원하지 않음.
longdesc URL 프레임에 보일 콘텐츠에 대한 설명을 담고 있는 페이지를 명시함.
HTML5에서는 더 이상 지원하지 않음.
ex) <frame longdesc="URL">

HTML5에서는 더 이상 지원하지 않음.
ex) <frame marginheight="pixels">

name 텍스트 프레임의 이름(name)을 명시함.

noresize noresize 사용자가 프레임의 크기를 조절할 수 있는지 여부를 명시함.
HTML5에서는 더 이상 지원하지 않음.
scrolling yes, no, auto 프레임에 스크롤바(scrollbar)를 표시할지 여부를 명시함.
HTML5에서는 더 이상 지원하지 않음.
